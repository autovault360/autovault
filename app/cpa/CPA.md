AutoVault360 CPA Portal - Developer Explanation
Purpose of the CPA Portal
The CPA Login is not a separate system from the main CRM. All data already exists inside the
Admin Dashboard, including Profit & Loss, Expenses, Vehicles, Payroll, State Tax, Deal Jackets,
Documents, and other financial records. The CPA Portal simply organizes all of this information into
a single accounting-focused dashboard so the CPA can efficiently review dealership activity by
month and year.
CPA Access Permissions
• Read-only access to dealership financial information.
• Can view all reports, financials, taxes, payroll, vehicles, and supporting documents.
• Can export reports and generate CPA packages.
• Can create and manage CPA Notes.
• Cannot edit, delete, modify, or create dealership records.
• Cannot modify vehicle data, expenses, payroll entries, or customer information.
Admin Access
Both Admin users can access the CPA Portal and see everything the CPA sees. Admins maintain
full edit permissions and are responsible for responding to CPA requests and updating dealership
records when necessary.
CPA Notes Workflow
The CPA can add notes anywhere throughout the CPA Portal. Examples:
• Missing receipt requests
• Missing deal jacket documents
• Payroll verification requests
• Sales tax questions
• Audit preparation notes
When a CPA adds a note: • Both Admin users can immediately see it.
• Admins can upload missing documents or correct records.
• CPA reviews updates but cannot directly edit dealership data.
Monthly Accounting Workflow
The CPA Portal is built around a monthly calendar selector. When the CPA clicks a month, the
system automatically displays: • Profit & Loss for that month
• Expenses for that month
• Vehicles sold and purchased
• Payroll and commissions
• State tax information
• Deal jacket activity
• CPA notes and reminders
• Supporting documents and exports
Final Objective
The CPA should be able to select any month and instantly review all dealership financial activity
required to prepare taxes, reconcile books, verify payroll, review commissions, and identify missing
documentation without requesting information through email or external systems.
